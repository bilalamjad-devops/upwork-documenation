# 🚀 Upwork Proposal Master Notes

## (Lesson: Reading Job Posts Creatively — Irfan Malik)

---

# Golden Rule

```text
70% Job Winning = Reading Job Description
30% Job Winning = Writing Proposal
```

Most freelancers focus on writing proposals.

Successful freelancers focus on understanding the client.

---

# 1. First Identify:

## WHO IS WRITING THE JOB POST?

Ask yourself:

```text
Who is behind this post?
```

Possible types:

### Technical Founder

Example:

```text
Terraform
AWS
Docker
Kubernetes
```

Approach:

```text
Use technical language.
```

---

### Non-Technical Client

Example:

```text
Need someone to automate my cloud infrastructure.
```

Approach:

```text
Avoid complex terminology.
Explain business benefits.
```

---

### Middleman / HR / Assistant

Example:

```text
We are looking for an AWS engineer.
```

Approach:

```text
Be simple.
Be professional.
Show confidence.
```

---

# 2. Mandatory Skills

These are:

```text
Must-have Skills
```

Example:

```text
AWS
Terraform
EC2
RDS
```

Question:

```text
Can I do this work?
```

If yes:

```text
Apply.
```

---

# 3. Advantageous Skills

These give you an edge.

Example:

```text
Docker
Azure
Kubernetes
Ansible
```

Even if they aren't your main skills:

```text
You must understand them.
```

Reason:

```text
Your growth stops when you completely ignore adjacent technologies.
```

---

# 4. Hidden Judgement Criteria

Sometimes clients write:

```text
Attention to detail
```

This is not random.

This means:

```text
I will judge your proposal
on attention to detail.
```

Examples:

* spelling mistakes
* copy-paste proposals
* missing requirements
* ignoring instructions

---

# 5. Reading Is The Real Skill

Most freelancers think:

```text
Proposal writing = skill
```

Actually:

```text
Reading the job post = skill
```

Because:

```text
Bad reading
=
Bad proposal
```

---

# 6. Extract The Client's Pain Point

Never ask:

```text
What technologies are needed?
```

Ask:

```text
What problem is the client trying to solve?
```

Example:

Client writes:

```text
Need Terraform engineer.
```

Hidden pain point:

```text
Need automation.
Need repeatability.
Need reliability.
Need less manual work.
```

---

# 7. Pain Point Formula

```text
Job Post
        ↓
Read Carefully
        ↓
Extract Pain Point
        ↓
Match Your Experience
        ↓
Write Proposal
```

---

# 8. Indirect Self-Praise

Never say:

```text
I am the best.
```

Instead say:

```text
I recently built a production-style
AWS environment using Terraform,
EC2 and RDS.
```

You are praising yourself indirectly.

---

# 9. Natural Conversation

Bad:

```text
Dear Sir,

I am a DevOps engineer with
excellent communication skills.
```

Good:

```text
I understand you're looking
for AWS infrastructure automation
using Terraform.
```

It should feel like:

```text
A natural conversation.
```
